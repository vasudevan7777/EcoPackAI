MODULE 2: DATA PROCESSING AND FEATURE ENGINEERING
=================================================

Overview:
This module processes raw material data and creates engineered features for
machine learning. It transforms the data into a format suitable for model training.

Main Script:
------------
data_processor.py - Central processing script that handles all transformations

Processing Steps:
-----------------

1. Data Loading
   Reads materials_data_100_records.csv from the data folder containing 100
   material records with 9 base features.

2. Material Type Encoding
   Converts categorical material types (Paper, Plastic, Bio-based, etc.) into
   numerical values using label encoding. This allows machine learning models
   to process the material categories.

3. Feature Engineering
   Creates three new calculated features:

   a) CO2 Impact Index
      Formula: co2_emission_score / cost_per_kg
      Purpose: Measures environmental impact relative to cost
      Higher values indicate worse environmental efficiency

   b) Cost Efficiency Index
      Formula: strength_mpa / cost_per_kg
      Purpose: Measures structural strength per unit cost
      Higher values indicate better value for money

   c) Material Suitability Score
      Formula: Weighted combination of multiple factors
      - 30% strength (normalized)
      - 30% biodegradability (normalized)
      - 20% recyclability (normalized)
      - 20% inverse CO2 emissions (normalized)
      Purpose: Overall material quality rating
      Range: 0 to 1 (higher is better)

4. Feature Normalization
   Normalizes five key features to 0-1 range:
   - strength_mpa
   - weight_capacity_kg
   - biodegradability_score
   - co2_emission_score
   - recyclability_percent

   Formula: (value - min) / (max - min)
   Purpose: Ensures all features are on same scale for ML models

Output:
-------
File: materials_data_processed.csv (saved in data folder)
Columns: 18 total
- Original 9 features
- 1 encoded feature (material_type_encoded)
- 3 engineered features
- 5 normalized features

Why This Matters:
-----------------
The engineered features help machine learning models better understand:
- Cost-effectiveness of materials
- Environmental impact trade-offs
- Overall material suitability

Normalized features ensure that:
- All values are comparable
- No single feature dominates due to scale
- Models train more efficiently

How to Use:
-----------
1. Ensure materials_data_100_records.csv exists in data folder
2. Run: python data_processor.py
3. Output file created: data/materials_data_processed.csv
4. File is ready for Module 3 (ML dataset preparation)

Verification:
-------------
Check that the output file contains 100 records and 18 columns.
All normalized features should have values between 0 and 1.
No missing values should be present.
